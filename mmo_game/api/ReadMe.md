玩家移动或者发起世界聊天时,客户端会分别向服务端发出msgId=3和msgId=2的消息,这部分代码是写在
client的源码（C#）里的,github只提供了打包后的exe,没有提供源码,所以不用管,本项目专注于使用zInx
框架实现本游戏世界的服务端逻辑。

服务端核心底层代码是这个mmoGoProject\zInx\znet\connection.go,监听协程会解包client端发来的消息
并将他们发给消息队列,消费协程会消费队列里的消息,解包出消息类型并由服务端上注册的路由逻辑来处理这些不同
类型的消息（处理逻辑就写在本api包下,主要就是一个世界聊天处理逻辑和一个玩家位置广播处理逻辑。）

客户端代码收到不同类型的客户端消息（如世界聊天，位置同步，玩家下线）时触发不同的业务逻辑,也是写在
客户端C#源码里的,无需深究。

客户端程序收到各类消息后如何显示在unity引擎上,也是C#源码里的逻辑,无需深究。
